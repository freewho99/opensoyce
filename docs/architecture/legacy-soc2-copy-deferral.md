**Date:** 2026-06-06

This document is the second Phase 3 implementation PR. It does not change any code. It records a deliberate deferral, locates the legacy copy precisely so it cannot be forgotten, and ties the cleanup to a future phase activation rather than fixing it ahead of the capability that would make it honest.

## 1. The decision

**Option 3 (defer to Phase 8) wins.** The legacy SOC 2 marketing copy on `/` and in the sidebar nav stays where it is today, with no rewrite, no removal, and no banned-substring exception. The copy becomes operational pressure for Phase 8 to actually ship.

The two rejected options:

- **Option 1 (quarantine)** — rejected because quarantining marketing copy without rewriting it preserves the false claim without the discipline to back it. Quarantine is a half-step that delays the honest decision.
- **Option 2 (rewrite)** — rejected because rewriting legacy marketing copy ahead of the capability that would justify it is exactly the drift the trust spine doctrine was built to prevent. The Trust Center's banned-substring vocabulary protects against marketing-first / capability-later claims. Rewriting now (e.g., into vaguer compliance copy) avoids the false claim but loses the operational signal that the capability is missing.

Option 3 keeps the tension visible. The legacy copy is honestly inherited debt; the capability is honestly absent; the resolution is honestly tied to Phase 8 actually shipping.

### 2.3 Guard compliance tab (`/guard?tab=compliance`)

The destination of the sidebar entry. Content on this tab is the next-level claim surface that the testimonial and nav entry imply OpenSoyce can back. This deferral applies to whatever copy lives there today, with the same rule: no rewrite, no removal, no banned-substring exception in this PR.

## 3. Why the deferral is correct

Three reasons, each grounded in prior doctrine.

### 3.1 The principle is encoded in code

From the [Trust Center sketch](./open-source-trust-center-sketch.md) §4 and the [roadmap integration](./open-soyce-roadmap-integration.md) §5 (banned-substring discipline):

> Capability + banned-substring exception ship in the same PR, never separately.

Rewriting the legacy copy in Phase 3 either keeps the SOC 2 claim (violating the rule) or removes it (separating the rewrite from the capability, also violating the rule). Deferring keeps the rule whole.

### 3.2 The roadmap already has the slot

[Roadmap integration](./open-soyce-roadmap-integration.md) §3 labels Phase 8 as **Blocked until evidence exists**. The legacy copy is not a Phase 3 problem; it is a Phase 8 *trigger*. Treating it as Phase 3 work pulls a Phase 8 decision forward at the cost of Phase 8 doctrine.

### 3.3 The Trust Center hygiene test already protects the launch surface

PR #55 added line-mode hygiene on `src/components/Layout.tsx` precisely to avoid catching the adjacent legacy SOC 2 entry as Trust-Center-adjacent debt. The hygiene boundary is now structural:

- The **new** Trust Center entry is hygiene-checked
- The **legacy** SOC 2 entry is acknowledged and excluded — by design
- Any **future** copy that drifts toward the legacy entry's tone fails the existing tests

Phase 8 lifts this exclusion by widening the Layout.tsx hygiene back to window-mode in the same PR that ships the capability — which is the cleanest signal possible that the legacy copy can no longer hide behind line-mode scoping.

## 5. What unblocks the deferral

The deferral lifts under exactly one condition: **Phase 8 ships an evidence layer that the legacy copy can honestly back.**

Phase 8 activation is gated by:

1. Phases 5 + 6 + 7 closeouts merged (per the roadmap integration §3 Phase 8 row).
2. A documented evidence-to-export mapping (`docs/architecture/evidence-export-mapping.md`) that names every export field and the live evidence that proves it.
3. A decision on what is exportable vs. what stays in the Trust Vault.

When Phase 8's first implementation PR ships, that PR atomically:

- Lifts the appropriate entries from `OPEN_SOURCE_TRUST_CENTER_BANNED_SUBSTRINGS` (`SOC 2` / `SOC2` if SOC 2 export ships; `Vanta` / `Drata` if those exports ship; etc.) per the roadmap integration §5 banned-substring discipline.
- Rewrites (or preserves verbatim, now legitimately) the three legacy copy locations in §2.
- Widens `src/components/Layout.tsx` hygiene back from line-mode to window-mode in `scripts/test-open-source-trust-center.mjs`.
- Removes this deferral document or updates its status to "Closed by Phase 8 PR #X."

Until that PR exists, the deferral stands. The legacy copy remains visible on `main`, with full knowledge that it is unbacked. That visibility is the operational pressure Phase 8 needs.
